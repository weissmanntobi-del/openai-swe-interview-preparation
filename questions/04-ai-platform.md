# AI Platform, Model Serving, and Applied AI

## 29. Model-Routing Gateway

Route requests across models based on capability, latency, cost, quality, availability, and policy.

Cover compatibility, fallback, experimentation, circuit breaking, data handling, and routing evaluation.

## 30. GPU Inference Scheduler

Schedule heterogeneous inference workloads on a shared accelerator fleet.

Cover batching, model loading, memory pressure, priorities, fairness, tail latency, utilization, and preemption.

## 31. Retrieval-Augmented Generation Platform

Design a multi-tenant RAG platform.

Cover ingestion, versioning, chunking, embeddings, filters, authorization, hybrid retrieval, reranking, citation support, and freshness.

## 32. Evaluation Platform

Design offline and online evaluation for prompts, models, agents, and RAG pipelines.

Cover datasets, graders, human review, statistical uncertainty, release gates, regressions, monitoring, and rollback.

## 33. Agent Execution Platform

Design a system for agents that call tools and run long-lived workflows.

Cover permissions, state, retries, human approval, loop detection, sandboxing, budget limits, and auditability.

## 34. Token-Streaming Service

Design reliable token streaming to web and mobile clients.

Cover transport, buffering, cancellation, reconnect, deduplication, final-state reconciliation, and usage accounting.

## 35. Prompt and Model Release Platform

Design controlled release of prompt, policy, and model changes.

Cover versioning, experiments, compatibility, evaluation gates, shadow traffic, canaries, and rollback.

## 36. AI Safety Policy Enforcement

Design a service that enforces dynamic policies before and after model execution.

Cover latency, fail-open versus fail-closed, policy versions, appeals, audit logs, privacy, and regional rules.

## 37. Semantic Cache

Design caching for model responses without returning contextually incorrect answers.

Cover cache key semantics, user isolation, freshness, model versions, confidence, invalidation, and evaluation.

## 38. Cost and Quality Control Plane

Design a platform that lets teams manage quality, latency, and spend across AI workloads.

Cover budgets, attribution, routing, alerts, quotas, optimization experiments, and executive reporting.
